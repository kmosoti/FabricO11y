# Requirements

This directory will hold versioned functional and quality requirements. Requirements should be
observable, traceable to a work package and ADR, and paired with a verification surface.

Initial requirement groups will cover:

- evidence admission and validation;
- ordering and replay determinism;
- corrections and maintained state;
- frontiers, provenance, and answer completeness;
- segment integrity and recovery;
- classification, redaction, retention, and audit;
- SDK parity and transport conformance.

Research notes and implementation ideas are not requirements until promoted here with acceptance
criteria.
